### üìã Functional Requirements

---

- [ ] The system shall generate topic-based performance analytics for each exam.
- [ ] The system shall allow users to filter analytics reports by exam topic.
- [ ] Only authorized users shall be able to access their own analytics reports.
- [ ] The analytics interface shall be user-friendly and support data visualization.
- [ ] Reports shall be exportable in common formats (e.g., PDF, CSV).

---

### üß© Use Case Scenario

**Actor:** Authorized User (e.g., student or instructor)

**Precondition:** User is authenticated and has exam results available.

**Steps:**
1. User logs in to the system.
2. User navigates to the Exam Analytics section.
3. User selects an exam to view analytics.
4. The system displays topic-based performance analytics for the selected exam.
5. User applies filters to view analytics by specific exam topics.
6. User exports or downloads the analytics report if needed.

**Postcondition:** User views and optionally exports topic-based analytics for their own exams.
